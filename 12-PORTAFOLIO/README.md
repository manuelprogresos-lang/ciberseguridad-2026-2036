# Portafolio profesional

Cada proyecto debe explicar:
- problema;
- objetivo;
- arquitectura;
- herramientas;
- decisiones;
- controles éticos;
- instalación;
- pruebas;
- resultados;
- limitaciones;
- mejoras futuras.

No publiques:
- credenciales;
- tokens;
- datos personales;
- IP reales;
- información empresarial;
- malware;
- código diseñado para abuso.
